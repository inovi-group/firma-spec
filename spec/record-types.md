# FIRMA record types

FIRMA defines eight record types. Every record in the ledger is exactly one of these types.

## Type codes

| Code | Name | Description | Immutability |
|---|---|---|---|
| `DR` | Decision Record | A governance decision made or mediated by FIRMA | Absolute |
| `ER` | Execution Record | Automated rule execution with outcome | Absolute |
| `AR` | Alert Record | Security or coherence alert generated by monitoring | Absolute |
| `KR` | Kintsugi Record | Healing event — breach documented alongside its repair | Absolute |
| `CPR` | Context Packet Record | Decision context received from an intelligence source | Absolute |
| `CR` | Constitutional Record | Rule creation, amendment, or retirement | Absolute + Owner signature |
| `AUD` | Audit Record | External audit access event | Absolute |
| `RR` | Reciprocity Record | Coherence score update for an actor pair | Absolute |

All record types are append-only. No type permits update or deletion.

## Decision Record (DR)

Captures a decision event with coherence scoring.

```json
{
  "record_type": "DR",
  "content": {
    "decision_id": "uuid",
    "declared_intent": { "action": "string", "parameters": {} },
    "observed_reality": { "action": "string", "parameters": {} },
    "contradiction_signals": {
      "temporal": 0.0,
      "behavioural": 0.0,
      "quantitative": 0.0,
      "semantic": 0.0
    },
    "contradiction_score": 0.0,
    "confidence": 0.0,
    "narrative": "string"
  }
}
```

## Execution Record (ER)

Records the outcome of an automated rule execution.

```json
{
  "record_type": "ER",
  "content": {
    "rule_id": "uuid",
    "trigger_event": "string",
    "execution_result": "SUCCESS | FAILURE | BLOCKED",
    "side_effects": []
  }
}
```

## Alert Record (AR)

Generated when monitoring detects an anomaly.

```json
{
  "record_type": "AR",
  "content": {
    "alert_class": "INFO | WARNING | CRITICAL | CONSTITUTIONAL_BREACH",
    "source": "string",
    "description": "string",
    "affected_actors": ["uuid"]
  }
}
```

## Kintsugi Record (KR)

Documents a breach alongside its remediation. Named after the Japanese art of repairing with gold — a healed record is more valuable than an unbroken one.

```json
{
  "record_type": "KR",
  "content": {
    "breach_record_hash": "hex string (reference only)",
    "remediation_action": "string",
    "actors_involved": ["uuid"],
    "coherence_score_before": 0.0,
    "coherence_score_after": 0.0
  }
}
```

## Context Packet Record (CPR)

Records a Decision Context Packet received from an intelligence source. See [dcp-protocol.md](./dcp-protocol.md).

## Constitutional Record (CR)

Creates or amends a governance rule. Requires chain owner signature for Absolute Rules.

```json
{
  "record_type": "CR",
  "content": {
    "rule_id": "uuid",
    "rule_version": "semver",
    "rule_type": "ABSOLUTE | CONDITIONAL | TEMPORAL | RECIPROCAL",
    "priority": 1,
    "trigger": { "type": "string", "conditions": [] },
    "action": { "type": "string", "parameters": {} },
    "supersedes": "uuid | null"
  }
}
```

## Audit Record (AUD)

Logs external audit access for accountability.

## Reciprocity Record (RR)

Updates the coherence score for an actor pair.

```json
{
  "record_type": "RR",
  "content": {
    "actor_pair": ["uuid", "uuid"],
    "coherence_score": 0.0,
    "sub_scores": {
      "responsiveness": 0.0,
      "consistency": 0.0,
      "reciprocity": 0.0,
      "trust_history": 0.0,
      "value_alignment": 0.0
    }
  }
}
```
